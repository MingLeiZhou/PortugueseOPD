# Repository structure

## Version-controlled content

- `src/`: all reproducible pipeline, diagnostic scenario, and QA smoke-test code.
- `docs/`: stable project, release, and QA documentation.
- `README.md`: user entry point, rebuild commands, and QA smoke-test entry points.
- `requirements.txt`: Python dependencies.
- `LICENSE`: source-code license.
- `DATA_LICENSE.md`: data redistribution boundary.
- `CITATION.cff`: citation metadata template.

## Local generated content

- `data/raw/`: source downloads generated by `src/download_datasets.py`.
- `data/metadata/`: source metadata and profiling artifacts.
- `data/processed/`: topology, benchmark, and Stage 1-5 release artifacts.
- `reports/`: generated Markdown reports.
- `figures/`: generated plots.
- `logs/`: execution logs.

These paths are excluded from ordinary Git. A future PT60-Candidate data release should be
deposited separately with a DOI under the E-REDES portal's stated CC BY 4.0 terms, retaining
publisher attribution, source identifiers and access dates, and modification notices.

## Canonical local release products

When present, the most important processed directories are:

- `data/processed/acpf_s16_backbone_core_depth6/`
- `data/processed/dcopf_s30_atpl_00147_parallel_equivalent/`
- `data/processed/dataset_release_stage1/`
- `data/processed/dataset_release_stage2/`
- `data/processed/dataset_release_stage3/`
- `data/processed/dataset_release_stage4/`
- `data/processed/dataset_release_stage5/`

Exploratory scenario outputs are not part of the repository release boundary.
